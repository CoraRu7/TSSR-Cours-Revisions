# Fiche Technique – Création d’un Monitor avec PRTG

## Objectif

Surveiller les connexions et le trafic réseau pour détecter anomalies, intrusions, ou saturation de bande passante.

---

## Concepts Clés

* **Monitoring** : Surveillance en temps réel d’un équipement ou d’un réseau.

  * Capteurs possibles : CPU, RAM, disque, carte réseau, services spécifiques.
* **But** : Analyser la bande passante, détecter les problèmes et optimiser les transferts.
* **Exemple** : Décaler l’envoi de sauvegardes pour éviter l’engorgement réseau.

---

## Installation PRTG Network Monitor (Windows Server 2016)

1. Télécharger PRTG depuis [Paessler](https://www.paessler.com/fr#products) avec clé licence.
2. Double-cliquer sur le `.exe`.
3. Sélectionner la langue et accepter le contrat de licence.
4. Saisir une adresse mail pour les notifications.
5. Choisir **installation rapide**.
6. Patienter pendant l’installation.
7. Le navigateur Internet Explorer s’ouvre automatiquement pour configurer PRTG.
8. Identifiants par défaut :

   * Login : `prtgadmin`
   * Mot de passe : `prtgadmin`
9. Accéder au **dashboard** : scan initial du réseau terminé.

## Méthode – Ajout d’un équipement distant et capteurs

**Exemple VM :**

* Serveur PRTG : WinServ-AD
* Équipement simulé : WinServ-Stock

### Étapes

1. **Ajouter un équipement**

   * `Équipements → Ajouter un équipement → Serveurs`
   * Remplir : Nom et IP → OK
2. **Ajouter un capteur Ping**

   * `Ajouter un capteur → Ping → Créer`
   * Attendre que le carré passe au vert.
3. **Ajouter un capteur Trafic réseau**

   * `Ajouter un capteur → Trafic réseau → Trafic (SNMP)`
   * Sélectionner la carte réseau de l’équipement → Créer
   * Attendre que le carré passe au vert.
4. **Surveillance**

   * Cliquer sur le capteur Ethernet → données toutes les 60 secondes.
   * Graphiques détaillés disponibles sur la droite du dashboard.
5. **Analyse du trafic**

   * Trafic normal : journée ou nuit planifiée.
   * Trafic en charge : vérifier les pics anormaux, surtout la nuit ou hors planification → peut signaler intrusion ou surcharge.

---

## Astuces et recommandations

* Synchroniser le temps entre les serveurs pour cohérence des logs et mesures.
* Utiliser les plugins PRTG (ex : Netflow) pour affiner l’analyse.
* PRTG = outil polyvalent : capteurs réseau + surveillance des serveurs, services et applications.
* Vérifier régulièrement le tableau de bord pour détecter anomalies.
* Comparer trafic normal vs trafic en charge pour identifier les problèmes de performance.

---

## Essentiel

* Poser des capteurs sur les interfaces réseau pour monitorer le trafic en temps réel.
* Adapter l’analyse selon le moment de la journée et la charge prévue.
* Identifier les anomalies rapidement pour intervenir avant que le réseau ne soit saturé.
* Surveillance interne et externe = informations précieuses pour la continuité et la sécurité.
