# **Fiche de révision : Mise en œuvre de la commutation et du routage**

## **Objectif**

Comprendre la commutation et le routage dans un réseau, apprendre à configurer un routeur et simuler un réseau avec Packet Tracer.

---

## **Concepts clés**

### **1. Principe de commutation**

* **Définition :** Transfert de données sous forme de paquets entre périphériques, uniquement vers le destinataire prévu (évite l’envoi inutile aux autres machines).
* **Fonctionnement :** Utilise les **adresses MAC** pour identifier les périphériques sur le réseau local (LAN).
* **Avantages :**

  * Optimisation de la bande passante
  * Transfert plus rapide et efficace

### **2. Méthodes de commutation**

1. **Commutation par datagramme** : Paquets contiennent l’adresse de destination complète (ex. emails).
2. **Commutation par circuit virtuel** : Crée un circuit logique entre émetteur et récepteur pour transferts réguliers (ex. appels téléphoniques).
3. **Commutation par routage à la source** : L’hôte source fournit toutes les informations nécessaires pour acheminer le paquet (ex. communications aérospatiales).

### **3. Différence entre commutateur et routeur**

| Équipement           | Fonction principale                   | Adresse utilisée |
| -------------------- | ------------------------------------- | ---------------- |
| Commutateur (Switch) | Transfert dans un LAN                 | MAC              |
| Routeur              | Transfert entre réseaux (LAN/WAN/MAN) | IP               |

### **4. Rôle du routeur**

* Permet la communication entre plusieurs réseaux.
* Acheminement des paquets entre périphériques de réseaux différents.
* Exemple conceptuel : comme la Poste qui fait circuler un colis entre différentes villes.

### **5. Packet Tracer**

* Simule des réseaux virtuels complexes sans matériel physique.
* Permet de construire, configurer et dépanner des réseaux LAN, WAN et PAN.

## **Configuration d’un routage avec Packet Tracer**

### **Étapes essentielles**

1. Définir le **hostname** du routeur
2. Configurer les **interfaces réseau**
3. Configurer la **table de routage**
4. Vérifier la **connectivité**

---

### **1. Définir le hostname**

* Onglet **Config** → modifier **Display Name** et **Hostname** (ex. `Router01`).

---

### **2. Configurer les interfaces réseau**

* **Définition :** Ports physiques ou logiques des périphériques avec une adresse IP unique.
* **Rôle :** Permettent le passage des paquets.
* **Procédure CLI pour une interface** :

```
Router> enable
Router# configure terminal
Router(config)# interface [nom_interface]      // ex: fa0/0
Router(config-if)# ip address [adresse_IP] [masque]  // ex: 192.168.1.1 255.255.255.0
Router(config-if)# no shutdown
Router(config-if)# end
```

* **Répéter** pour chaque interface à configurer.
* Vérifier que les LEDs passent du rouge au vert pour signaler qu’elles sont actives.

---

### **3. Configurer la table de routage**

* **Définition :** Liste des réseaux connus par le routeur, utilisée pour déterminer le meilleur chemin pour les paquets.
* **Afficher la table de routage :**

```
show ip route
```

* **Ajouter une route statique** :

```
Router> enable
Router# configure terminal
Router(config)# ip route [network_IP] [mask_IP] [Next_Hop_IP]
Router(config)# end
Router# write         // Enregistre la configuration
```

* **Explications des paramètres :**

  * `network_IP` : réseau destination
  * `mask_IP` : masque du réseau destination
  * `Next_Hop_IP` : IP du prochain saut pour atteindre le réseau

---

### **4. Vérifier la connectivité**

* Sur un PC : **Desktop → Command Prompt → ping [IP du périphérique distant]**
* Si tous les paquets sont reçus, le routage est fonctionnel.

---

## **Résumé des commandes CLI importantes**

```
enable
configure terminal
interface [nom_interface]
ip address [IP] [masque]
no shutdown
end
show ip route
ip route [network_IP] [mask_IP] [Next_Hop_IP]
write
```

---

## **Notions transposables à d’autres situations**

* La structure de configuration est la même quel que soit le réseau : **hostname → interfaces → table de routage → test**
* Les interfaces doivent toujours avoir des **IP uniques et activées**.
* La table de routage peut être utilisée pour tout type de réseau (LAN, WAN, MAN).
* Packet Tracer permet de tester toute configuration avant déploiement physique.
* La logique de commutation reste identique, quel que soit le type de périphérique ou la topologie réseau.
