# Fiche – Création et organisation des UO, utilisateurs et groupes

## Objectif
Savoir créer et organiser les **unités organisationnelles (UO)**, les **utilisateurs** et les **groupes** dans Active Directory, afin de gérer efficacement les droits et l’administration.

---
<details>
<summary>

## 1️⃣ Unité d’Organisation (UO)
</summary>

### Définition
- **UO (OU, Organizational Unit)** : conteneur dans l’Active Directory permettant d’organiser les objets (utilisateurs, groupes, ordinateurs).  
- Rôle : faciliter l’administration, déléguer des droits et appliquer des stratégies (GPO).  
- Une UO peut contenir :
  - Plusieurs **sous-UO**
  - Des **objets** uniques (utilisateurs, groupes, ordinateurs)

### Schéma conceptuel UO

Domaine SRV.STUDI
│
├─ UO Comptabilité
│ ├─ Utilisateur: alice
│ ├─ Utilisateur: bob
│ └─ Groupe: Comptables
│
├─ UO IT
│ ├─ Utilisateur: dami
│ └─ Groupe: Administrateurs IT
└─ UO Marketing
├─ Utilisateur: claire
└─ Groupe: Marketers

### Bonnes pratiques
- **Protéger le conteneur contre suppression accidentelle** : cocher toujours cette option.  
- Nommer les UO de manière claire et cohérente.

## 2️⃣ Création d’une UO
</summary>

### Étapes
1. Gestionnaire de serveur → Outils → **Utilisateurs et ordinateurs Active Directory**  
2. Clic droit sur le domaine → **Nouveau → Unité d’organisation**  
3. Nommer l’UO → **OK**  

### Complément
- Ne pas décocher « Protéger contre suppression accidentelle ».  
- Les UO peuvent contenir sous-UO, utilisateurs et groupes.

</details>

<summary>

## 3️⃣ Création d’un utilisateur dans une UO
</summary>

### Étapes
1. Sélectionner l’UO → clic droit → **Nouveau → Utilisateur**  
2. Remplir les informations :
   - Nom complet, prénom, nom de connexion  
3. Mot de passe :
   - Saisir le mot de passe
   - Options de mot de passe : changer au prochain logon, mot de passe non expirant, etc.  
4. Vérifier les informations → **Terminer**  

### Notes
- L’utilisateur peut se connecter avec son **login Windows** ou son **email** (si configuré).  
- Répéter pour chaque nouvel utilisateur.

<summary>

## 4️⃣ Création d’un groupe dans une UO
</summary>

### Étapes
1. Sélectionner l’UO → clic droit → **Nouveau → Groupe**  
2. Nommer le groupe → options par défaut :
   - **Étendue** : Globale (recommandé)  
   - **Type** : Sécurité ou Distribution  
3. Ajouter des membres au groupe (utilisateurs ou autres groupes)  

### Tableau de configuration

| Paramètre       | Recommandation                     | Description |
|-----------------|----------------------------------|-------------|
| Étendue         | Globale                           | Les membres peuvent être du même domaine |
| Type            | Sécurité                          | Permet la gestion des permissions |
| Protection      | Activée                           | Empêche la suppression accidentelle |

### Complément
- On peut déplacer des utilisateurs entre groupes ou supprimer des membres.  
- Un groupe peut contenir plusieurs utilisateurs ou sous-groupes.

## 6️⃣ Bonnes pratiques
</summary>

- Toujours **protéger les UO et groupes contre suppression**.  
- Définir une **convention de nommage** claire pour les UO, utilisateurs et groupes.  
- Créer des **groupes pour les permissions**, ne jamais donner les droits directement aux utilisateurs.  
- Vérifier les droits et la hiérarchie avant d’appliquer des **GPO**.  
- Documenter toutes les créations pour l’audit et la maintenance.
