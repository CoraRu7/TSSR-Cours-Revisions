# Fiche 1 – Concepts clés : Configuration Active Directory
</summary>

## Objectif
Comprendre le rôle d’un contrôleur de domaine et savoir configurer l’Active Directory (AD DS) pour qu’il soit opérationnel et sécurisé.

## Définitions
- **Active Directory (AD DS)** : répertoire central qui organise et gère les utilisateurs, ordinateurs, groupes et ressources d’un réseau.  
- **Contrôleur de domaine (DC)** : serveur qui exécute AD DS, vérifie les identités et coordonne les accès aux ressources réseau.  
- **Domaine** : unité logique dans AD, regroupant utilisateurs, ordinateurs et groupes.  
- **Forêt et NetBIOS** : hiérarchies permettant de structurer les domaines et faciliter la compatibilité avec certains services.

## Astuces de sécurité
- AD est une cible privilégiée pour les attaques ; une configuration correcte est essentielle.  
- Les erreurs de configuration deviennent critiques au fur et à mesure que le domaine grandit.  
- AD est le noyau de l’infrastructure informatique, la première couche de sécurité et d’automatisation.

# Fiche 2 – Promotion du serveur en contrôleur de domaine
</summary>

## Objectif
Transformer un serveur Windows avec AD installé en contrôleur de domaine fonctionnel.

## Étapes détaillées
1. **Accès à l’assistant**  
   - Gestionnaire de serveur → Tableau de bord → Cliquer sur le drapeau jaune → « Promouvoir ce serveur en contrôleur de domaine ».  

2. **Configuration du domaine**  
   - Choisir **Ajouter une nouvelle forêt**.  
   - Nommer le domaine racine, par exemple `STUDI.SRV`.  
   - Cliquer sur **Suivant**.  

3. **Options du contrôleur de domaine**  
   - Définir un mot de passe pour le **mode de restauration des services d’annuaire (DSRM)**.  
   - Sélectionner le niveau fonctionnel du domaine (laisser par défaut si toutes les machines sont Windows Server 2016 ou ultérieur).  

4. **Options DNS**  
   - Laisser par défaut pour créer la délégation DNS.  
   - Cliquer sur **Suivant**.  

5. **Options supplémentaires et NetBIOS**  
   - Vérifier le nom NetBIOS attribué automatiquement.  
   - Modifier si nécessaire, puis **Suivant**.  

6. **Chemins d’accès**  
   - Emplacements par défaut pour : base de données AD, fichiers journaux, SYSVOL.  
   - Cliquer sur **Suivant**.  

7. **Examiner les options**  
   - Vérifier toutes les configurations avant l’installation.  
   - Cliquer sur **Suivant**.  

8. **Vérification et installation**  
   - L’assistant vérifie la configuration requise.  
   - Cliquer sur **Installer**.  
   - Le serveur redémarre automatiquement.  

9. **Connexion après redémarrage**  
   - Se connecter avec le compte administrateur du domaine : `STUDI\Administrateur`.  
   - Vérifier dans Gestionnaire de serveur → Serveur local → Nom du domaine.

# Fiche 3 – Bonnes pratiques de configuration AD
</summary>

## Sécurité et gestion
1. **Renommer l’administrateur de domaine** pour éviter les attaques ciblées (`AdminContosoAD`).  
2. **Mot de passe fort pour l’administrateur et les utilisateurs**.  
3. **Informations d’identification dédiées pour l’informatique**, séparées de la direction.  
4. **Attribuer les bonnes autorisations aux administrateurs**, éviter qu’un utilisateur ne soit au-dessus du niveau de domaine.  
5. **Configurer les GPO** avec parcimonie et organisation claire.  
6. **Activer la corbeille AD** pour restaurer facilement des objets supprimés.  
7. **Au moins deux contrôleurs de domaine** pour éviter les pannes critiques.  
8. **Supprimer les éléments obsolètes** (utilisateurs et ordinateurs inutilisés).  
9. **Ne rien installer sur un contrôleur de domaine** : ni logiciel tiers, ni applications.  
10. **Définir une convention de nommage claire** pour utilisateurs, groupes, serveurs et appareils.  
11. **Maintenir les serveurs à jour** avec les patchs Windows.  
12. **Auditer les changements AD** pour savoir qui modifie quoi, prévenir les problèmes de sécurité.

# Fiche 4 – Résumé pratique
</summary>

- **AD DS = Base + services pour gérer le domaine.**  
- **DC = Serveur qui exécute le domaine et sécurise les accès.**  
- Promotion du serveur en DC = transformation du serveur avec AD installé en serveur actif pour gérer le domaine.  
- **Vérification** : connexion avec `domaine\administrateur` et affichage du nom de domaine dans le Gestionnaire de serveur.  
- **Sécurité** : changement de nom admin, mot de passe fort, GPO adaptées, au moins deux DC, audit, patchs réguliers.
