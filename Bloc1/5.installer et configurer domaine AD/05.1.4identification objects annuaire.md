# Fiche 1 – Concepts clés : ADSI Edit et Active Directory
</summary>

## Objectif
Savoir identifier, gérer et modifier les objets de l’Active Directory à l’aide de l’outil ADSI Edit.

## Définitions
- **ADSI Edit** : éditeur d’annuaires Microsoft permettant de visualiser et modifier tous les attributs des objets dans Active Directory.  
- **Objet AD** : élément stocké dans l’Active Directory (utilisateur, groupe, ordinateur, etc.).  
- **Attribut** : propriété spécifique d’un objet AD, par ex. `displayName`, `sAMAccountName`, etc.  
- **AD DS** : Active Directory Domain Services, service principal d’AD pour la gestion centralisée des utilisateurs et ordinateurs.  
- **Domaine, Arbre, Forêt** :  
  - Domaine : étendue administrative avec des objets liés (utilisateurs, ordinateurs).  
  - Arbre : regroupement de plusieurs domaines.  
  - Forêt : combinaison d’arbres ; permet les relations de confiance entre domaines différents.  

## Importance
- Permet la gestion fine de tous les objets AD.  
- Accès à des attributs invisibles depuis les consoles classiques (ADUC, Sites, Services).  
- Utile pour administrateurs et éditeurs de logiciels utilisant l’annuaire.

# Fiche 2 – Connexion et navigation dans ADSI Edit
</summary>

1. **Accéder à ADSI Edit**  
   - Gestionnaire de serveur → Outils → « Modification ADSI ».  

2. **Créer une connexion**  
   - Clic droit → Connexion → entrer le nom du domaine (`SRV.STUDI`)  
   - Laisser les options par défaut → OK  

3. **Explorer l’annuaire**  
   - Dans le volet gauche, cliquer sur le domaine (`SRV.STUDI`)  
   - Accès aux partitions, groupes, comptes utilisateurs et ordinateurs  

4. **Exemple de partition**  
   - `CN=Users` → toutes les sessions créées précédemment dans AD

# Fiche 3 – Gestion des objets et attributs
</summary>

1. **Modification d’un objet**  
   - Clic droit sur une session → Propriétés  
   - Tous les attributs sont modifiables  

2. **Filtrer les attributs**  
   - Filtrer → décocher « Afficher uniquement les attributs qui ont des valeurs »  
   - Permet de retrouver un attribut absent dans la liste par défaut  

3. **Modification d’un attribut**  
   - Exemple : `displayName DAM`  
   - Étapes : Modifier → Entrer le nouveau nom → OK → Appliquer  

4. **Vérification**  
   - Centre d’administration AD → Chercher l’utilisateur → Propriétés  
   - Le `Nom complet` reflète le changement

# Fiche 4 – Bonnes pratiques avec ADSI Edit
</summary>

- Toujours sauvegarder l’AD avant toute modification.  
- Ne pas modifier les attributs critiques si non nécessaire (`objectGUID`, `SID`).  
- Utiliser ADSI Edit pour les tâches avancées introuvables dans ADUC ou autres consoles.  
- Tester les changements sur un environnement de test avant production.  
- Filtrer les attributs pour éviter les erreurs et confusions.  
- Documenter toute modification apportée pour l’audit et la conformité.

# Fiche 5 – Résumé pratique
</summary>

- **ADSI Edit** : outil puissant pour identifier et gérer tous les objets et attributs de l’AD.  
- **Processus clé** : Outils → ADSI Edit → Connexion au domaine → Navigation → Modification → Vérification.  
- **Utilité principale** : accès complet à l’annuaire, gestion centralisée, modifications avancées.  
- **Sécurité** : prudence sur les objets sensibles, sauvegarde et documentation systématique.
