# **Fiche de cours : Émulation IOS Cisco sous Packet Tracer**

## **1. Émulation réseau et Packet Tracer**

### Définition

* **Émulation** : reproduire le comportement d’un système sur une plateforme différente.
* Permet de tester un réseau complet **sans matériel physique**.

### Utilité

* Tester des configurations et topologies.
* Apprendre à configurer des équipements Cisco.
* Simuler des situations réelles pour la formation et le dépannage.

### **Packet Tracer**

* Logiciel de simulation réseau Cisco.
* Permet :

  * Concevoir, configurer, dépanner un réseau LAN/MAN/WAN.
  * Interagir avec routeurs, switches, PC, serveurs.
  * Visualiser le trafic réseau en temps réel ou en simulation.

**Avantages :**

* Léger, fonctionne sur la plupart des PC.
* Utilisé pour les certifications Cisco CCNA & CCNP.

## **2. IOS Cisco**

### Définition

* **IOS (Internetwork Operating System)** : OS Cisco pour routeurs et switches.
* CLI (interface ligne de commande) pour :

  * Configurer interfaces et adresses IP.
  * Gérer routage et commutation.
  * Sécuriser équipements réseau.

### Différence IOS vs iOS

* **IOS Cisco** → Internetwork Operating System
* **iOS Apple** → iPhone Operating System

### Modes de fonctionnement IOS

| Mode                    | Prompt                 | Utilité                                   |
| ----------------------- | ---------------------- | ----------------------------------------- |
| Utilisateur (User EXEC) | `Router>`              | Commandes basiques, diagnostic            |
| Privilégié (Enable)     | `Router#`              | Commandes avancées, accès à configuration |
| Configuration globale   | `Router(config)#`      | Modifier configuration du routeur         |
| Configuration interface | `Router(config-if)#`   | Configurer une interface spécifique       |
| Configuration ligne     | `Router(config-line)#` | Configurer accès console/VTY (Telnet/SSH) |

**Commandes passage modes :**

```text
Router> enable
Router# configure terminal
Router(config)# interface GigabitEthernet0/0
Router(config-if)# exit
Router(config)# exit
Router# disable
Router>
```

## **4. Déploiement et connexion de base**

### Ajouter équipements

* 1 commutateur, 3 PC, routeur.
* Vérifier alimentation.

### Types de câbles

| Connexion         | Câble            |
| ----------------- | ---------------- |
| PC → Switch       | Droit (Ethernet) |
| Switch → Routeur  | Croisé           |
| Routeur → Routeur | Série            |

### Vérification

```text
show ip interface brief
ping <adresse_IP>
```

## **5. Commandes de configuration de base**

### Nom du routeur et bannières

```text
hostname RouterParis
banner motd #Accès réservé aux admins#
```

### Interface et IP

```text
interface GigabitEthernet0/0
ip address 192.168.1.1 255.255.255.0
no shutdown
exit
```

### Sauvegarde

```text
copy running-config startup-config
```

### Mot de passe Enable et console

```text
enable secret AdminSecure2024
line console 0
password adminpass
login
exit
```

### Accès distant sécurisé (SSH)

```text
hostname SecureRouter
ip domain-name example.com
crypto key generate rsa
line vty 0 4
transport input ssh
password sshpass
login
exit
```

## **6. VLAN et segmentation**

### Créer VLAN

```text
vlan 10
name Comptabilité
exit
```

### Affecter port au VLAN

```text
interface fa0/1
switchport mode access
switchport access vlan 10
exit
```

### Trunk et VTP

```text
interface gi0/1
switchport mode trunk
switchport trunk allowed vlan 10,20,30
exit

vtp domain MonReseau
vtp mode server
vtp password secret123
```

## **7. Routage inter-VLAN et OSPF**

### Sous-interfaces et routage

```text
interface gi0/0.10
encapsulation dot1Q 10
ip address 192.168.10.1 255.255.255.0
no shutdown
exit

interface gi0/0.20
encapsulation dot1Q 20
ip address 192.168.20.1 255.255.255.0
no shutdown
exit
```

### OSPF

```text
router ospf 1
network 192.168.10.0 0.0.0.255 area 0
network 192.168.20.0 0.0.0.255 area 0
exit
```

## **8. DHCP**

### Configuration pools

```text
ip dhcp excluded-address 192.168.10.1 192.168.10.10
ip dhcp pool VLAN10
network 192.168.10.0 255.255.255.0
default-router 192.168.10.1
dns-server 8.8.8.8
exit

ip dhcp excluded-address 192.168.20.1 192.168.20.10
ip dhcp pool VLAN20
network 192.168.20.0 255.255.255.0
default-router 192.168.20.1
dns-server 8.8.8.8
exit
```

## **9. Redondance (HSRP, VRRP, GLBP)**

### HSRP (Cisco)

```text
interface gi0/1
standby 1 ip 192.168.1.254
standby 1 priority 110
standby 1 preempt
```

* IP virtuelle : 192.168.1.254
* Priorité plus haute → routeur actif
* Preempt → basculement automatique

## **10. Sécurité avec ACL**

### ACL standard

```text
access-list 10 permit 192.168.1.0 0.0.0.255
interface gi0/1
ip access-group 10 in
```

### ACL étendue

```text
access-list 101 deny tcp any host 192.168.1.100 eq 23
access-list 101 permit ip any any
interface gi0/1
ip access-group 101 in
```
