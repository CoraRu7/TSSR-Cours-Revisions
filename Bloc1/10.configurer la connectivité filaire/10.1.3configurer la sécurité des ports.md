# **Fiche de révision : Sécurisation des ports sur un commutateur (Packet Tracer)**

## **Objectif**

Apprendre à sécuriser les ports d’un switch de manière **statique, dynamique et rémanente** sous Packet Tracer, avec les commandes CLI Cisco.

---

## **1. Pourquoi sécuriser les ports ?**

* Empêche les intrusions et attaques réseau.
* Limite le nombre d’adresses MAC sur chaque port.
* Protège la configuration du switch et les données réseau.

**⚠ Attention** : si les ports ne sont pas sécurisés, un pirate peut :

* Scanner le réseau
* Lancer des attaques (spoofing, flooding, etc.)

---

## **2. Types d’adresses MAC sécurisées**

| Type                   | Description                                                     | Persistance                                                |
| ---------------------- | --------------------------------------------------------------- | ---------------------------------------------------------- |
| **Statique**           | Configurée manuellement                                         | Ajoutée à la table et au running-config                    |
| **Dynamique**          | Apprise automatiquement                                         | Stockée uniquement dans la table, disparaît au redémarrage |
| **Rémanente (sticky)** | Apprise automatiquement puis sauvegardée dans le running-config | Persistante même après redémarrage si sauvegardée          |

**Remarque sur les adresses rémanentes :**

* `switchport port-security mac-address sticky` transforme les adresses dynamiques en rémanentes.
* `no switchport port-security mac-address sticky` arrête l’apprentissage rémanent mais garde les adresses dans la table.
* Les adresses rémanentes non sauvegardées disparaissent au redémarrage du switch.

---

## **3. Configuration de la sécurité des ports**

### **3.1 Configuration de base sur une interface**

```bash
S1> enable
S1# configure terminal
S1(config)# interface fa0/18         # Choisir l’interface à sécuriser
S1(config-if)# switchport mode access
S1(config-if)# switchport port-security
```

---

### **3.2 Adresse MAC statique**

```bash
S1> enable
S1# configure terminal
S1(config)# interface fa0/5
S1(config-if)# switchport mode access
S1(config-if)# switchport port-security
S1(config-if)# switchport port-security mac-address <MAC_ADDRESS>
S1(config-if)# end
```

* Stockée dans la table et dans le running-config.
* Définie manuellement.

---

### **3.3 Adresse MAC dynamique**

```bash
S1> enable
S1# configure terminal
S1(config)# interface fa0/5
S1(config-if)# switchport mode access
S1(config-if)# switchport port-security
S1(config-if)# end
```

* Apprise automatiquement.
* Disparaît au redémarrage du switch.

---

### **3.4 Adresse MAC rémanente (sticky)**

```bash
S1> enable
S1# configure terminal
S1(config)# interface fa0/5
S1(config-if)# switchport mode access
S1(config-if)# switchport port-security
S1(config-if)# switchport port-security maximum 10      # Limite à 10 adresses MAC
S1(config-if)# switchport port-security mac-address sticky
S1(config-if)# end
```

* Apprise automatiquement et sauvegardée dans le running-config.
* Persistante après redémarrage si sauvegardée.

---

## **4. Commandes CLI de vérification**

* Vérifier la sécurité d’un port :

```bash
switch# show port-security interface <id_interface>
```

* Vérifier toutes les adresses MAC sécurisées :

```bash
switch# show port-security
```

---

## **5. Désactivation des ports inutilisés**

```bash
S1(config)# interface range fa0/10-24
S1(config-if-range)# shutdown
```

* Protège contre l’accès non autorisé.
* Réduit les risques de vulnérabilité réseau.

---

## **6. Points importants à retenir**

* Toujours mettre les interfaces en **mode access** avant de sécuriser.
* Les adresses MAC **dynamiques** disparaissent au redémarrage, sauf si sticky.
* `switchport port-security maximum <n>` limite le nombre d’adresses MAC.
* Toujours vérifier la configuration avec `show port-security`.
* Pour sécuriser plusieurs ports : `interface range fa0/1-5`.
* Les ports inutilisés doivent être **shutdown**.

---

## **7. Schéma récapitulatif**

```
Port Access Mode
  │
  ├─ Statique : mac-address <MAC>
  │
  ├─ Dynamique : apprise auto, disparaît au reboot
  │
  └─ Rémanente (Sticky) : apprise auto + sauvegardée (persistante)
```
