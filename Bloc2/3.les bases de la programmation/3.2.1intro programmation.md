# **Introduction à la programmation**

## **Objectif**

Comprendre les bases de la programmation et de l’algorithmie pour créer des programmes informatiques capables de réaliser des tâches spécifiques.

---

## **Concepts clés**

### **1. Contexte**

* La programmation est essentielle dans tous les domaines informatiques : développement logiciel, sites web, analyse de données, etc.
* Fondements : **algorithmes**, **langages de programmation**, **processus de création de programmes**.

---

### **2. Qu’est-ce que la programmation ?**

* **Programmation** : donner des instructions à un ordinateur pour qu’il effectue des tâches spécifiques.
* **Programme** : ensemble d’instructions (code) qui accomplit une tâche précise.
* **Exemple** : un programme = recette de cuisine, chaque étape correspond à une instruction.

---

### **3. Rôle du programmeur**

* Transforme une idée en instructions logiques.
* Utilise langages, bibliothèques et frameworks pour écrire du code clair et structuré.
* Fonctionnalités clés pour un code lisible :

  * Indentations, commentaires, noms de variables significatifs
  * Documentation, fonctions réutilisables, gestion des espaces
  * Utilisation de constantes, bibliothèques, conventions, éviter la duplication
  * Organisation du code en blocs logiques

---

### **4. Historique et premiers langages**

| Période   | Langage         | Caractéristiques                                                  |
| --------- | --------------- | ----------------------------------------------------------------- |
| 1940-1950 | Langage machine | Instructions binaires (0 et 1), très bas niveau                   |
| 1950-1960 | Assembleur      | Remplace binaire par mnémoniques, lié à l’architecture matérielle |

* Exemple : **Baby / Manchester Mark 1** (1948) : 128 octets, 700 opérations/sec.

---

### **5. Étapes de création d’un programme**

1. **Comprendre le problème** : définir ce que doit faire l’ordinateur.
2. **Planifier** : création de l’algorithme.
3. **Écrire le code** : instructions dans un langage de programmation.
4. **Tester / Déboguer** : corriger les erreurs.
5. **Compiler** : traduire le code en langage machine exécutable.

---

### **6. Concepts clés**

#### **Algorithme**

* Séquence d’instructions logiques pour résoudre un problème.
* Indépendant du langage de programmation.
* Exemple : algorithme pour préparer un sandwich (étapes précises à suivre).

#### **Code**

* Traduction d’un algorithme dans un langage compréhensible par l’ordinateur.
* Permet l’exécution automatique et précise des tâches.

#### **Compilateur**

* Traduit le code source en code machine exécutable.
* Vérifie syntaxe, optimise et produit le fichier exécutable.

---

### **7. Programmation de haut et bas niveau**

* **Haut niveau** : proche du langage humain (ex. Python, Java, C#). Plus simple à utiliser.
* **Bas niveau** : proche du langage machine (ex. assembleur, langage machine). Plus complexe, contrôle précis du matériel.
* Utilisation combinée possible selon les besoins.

---

### **8. Fonctionnement global**

* Le processeur lit les instructions stockées en mémoire.
* Exécute les opérations, traite les données et produit le résultat.
* Le code source → compilateur → code machine → exécution par le processeur.

---

### **9. Essentiel**

* La programmation consiste à écrire du code pour créer des programmes capables de résoudre des problèmes.
* Importance historique : des langages machine et assembleur aux langages modernes de haut niveau.
* Processus de création : **comprendre → planifier → coder → tester → compiler**.
* Compétences nécessaires : logique, rigueur et créativité pour résoudre des problèmes informatiques.
