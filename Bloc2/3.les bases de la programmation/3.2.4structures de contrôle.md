# Les structures de contrôle

## Objectif

Comprendre et manipuler les structures de contrôle (conditions et branchements) pour gérer le flux d’exécution d’un programme.

## Concepts clés

### 1. Principes généraux

* **Expression booléenne** : évalue une condition à **True** (vrai) ou **False** (faux).

  * Opérateurs de comparaison : `==`, `<`, `>`, `<=`, `>=`, `!=`.
* **Test conditionnel** : exécuter une action uniquement si la condition est vraie.

### 2. Les tests conditionnels

#### a) Si (if)

* Syntaxe générale :

  ```
  if (condition) {
      // actions si vrai
  }
  ```
* Exemple :

  ```
  IF (N >= 10) Afficher "reçu"
  ```

#### b) Sinon (else)

* Permet de définir une alternative si la condition est fausse.
* Syntaxe :

  ```
  if (condition) {
      // actions si vrai
  } else {
      // actions si faux
  }
  ```
* Exemple :

  ```
  IF (S >= M) Autoriser la transaction
  ELSE Refuser la transaction
  ```

#### c) Sinon si (else if)

* Permet de gérer plusieurs cas successifs.
* Syntaxe :

  ```
  if (condition1) { ... }
  else if (condition2) { ... }
  else { ... }
  ```
* Exemple : Mention d’un examen :

  ```
  IF (N < 10) Afficher "non obtenu"
  ELSE IF (N < 12) Afficher "mention passable"
  ELSE IF (N < 14) Afficher "mention assez bien"
  ELSE IF (N < 16) Afficher "mention bien"
  ELSE Afficher "mention très bien"
  ```

### 3. Selon (switch…case)

* Permet de comparer une variable à plusieurs valeurs possibles.

* Syntaxe générale :

  ```
  switch (variable) {
      case valeur1:
          // actions
          break;
      case valeur2:
          // actions
          break;
      default:
          // actions si aucune correspondance
  }
  ```

* **Break** : termine l’exécution du `case` pour éviter l’exécution des suivants.

* **Default** : actions à exécuter si aucun `case` ne correspond.

* Exemple : température :

  ```
  SWITCH (T)
  CASE 37: Afficher "va bien", BREAK
  CASE 38: Afficher "légèrement fiévreux", BREAK
  CASE 39: Afficher "vraiment fiévreux", BREAK
  CASE 40: Afficher "très fiévreux", BREAK
  ```

### 4. Différences if…else if…else vs switch

| Critère         | if…else if…else                                   | switch…case                                          |
| --------------- | ------------------------------------------------- | ---------------------------------------------------- |
| Comparaison     | Plusieurs types (>, <, ==…)                       | Test d’égalité uniquement                            |
| Élément comparé | Variables ou expressions multiples                | Une seule variable                                   |
| Exécution       | Seule l’action correspondant au test est exécutée | Tous les `case` suivants s’exécutent si break absent |

## Commandes / Notes

* `var_dump(expression)` en PHP : affiche le type et la valeur d’une expression (utile pour vérifier les booléens).
* `echo` : afficher du texte à l’écran.
* `readline()` : demander une saisie utilisateur.
