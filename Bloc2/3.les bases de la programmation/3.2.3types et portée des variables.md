# **Types et portées des variables**

## **Objectif**

Comprendre les types de données, le typage, et la portée des variables afin de mieux structurer et sécuriser un programme.

---

## **Concepts clés**

### **1. Types de données**

* **Entier (int / ENTIER)** : nombres sans virgule (positifs, négatifs ou nuls)
* **Flottant / Réel (float / FLOTTANT)** : nombres avec virgule
* **Caractère (char / CARACTERE)** : un seul symbole
* **Chaîne (string / CHAINE)** : séquence de caractères
* **Booléen (bool / BOOLEEN)** : vrai ou faux
* **Tableaux / array** : collection de valeurs du même type accessibles par un indice
* **Objet / object** : instance d’une classe avec données et méthodes

---

### **2. Portée des variables**

* **Locale** : déclarée à l’intérieur d’une fonction / bloc, accessible uniquement à l’intérieur
* **Globale** : déclarée en dehors de toute fonction, accessible partout dans le programme
* **Bonne pratique** : privilégier la portée locale pour réduire les erreurs et contrôler l’accès aux données

---

### **3. Langages typés vs non typés**

| Caractéristique | Langages typés        | Langages non typés             |
| --------------- | --------------------- | ------------------------------ |
| Déclaration     | Type explicite requis | Type déterminé automatiquement |
| Vérification    | Compilation           | Exécution                      |
| Flexibilité     | Faible                | Élevée                         |
| Sécurité        | Plus sûr              | Moins sûr, erreurs possibles   |
| Performance     | Souvent meilleure     | Conversion à l’exécution       |
| Exemples        | Java, C++, C#         | Python, JavaScript, PHP        |

* **Langages typés** : rigueur, sûreté de typage, meilleures performances
* **Langages non typés** : flexibilité, syntaxe concise, mais risques d’erreurs

---

### **4. Typage statique vs dynamique**

* **Statique** : type de variable fixé à la déclaration, détecte les erreurs avant exécution
* **Dynamique** : type déterminé à l’exécution, plus flexible, mais plus de risques d’erreurs
* **Optionnel** : certains langages (TypeScript, Python avec annotations) permettent un typage hybride

---

### **5. Déclaration de variables en pseudo-code**

**Avec typage (langages typés)**

```pseudocode
DEBUT
ENTIER x <- 10
FLOTTANT y <- 3.14
CHAINE message <- "Bonjour"
BOOLEEN estVrai <- Vrai
FIN
```

**Sans typage (langages non typés)**

```pseudocode
DEBUT
x <- 10
y <- 3.14
message <- "Bonjour"
estVrai <- Vrai
FIN
```

---

### **6. Exemples d’opérations**

**Typé :**

```pseudocode
ENTIER a <- 5
ENTIER b <- 20
ENTIER somme <- a + b
REEL moyenne <- (REEL)somme / 2.0
```

**Non typé :**

```pseudocode
a <- 5
b <- 20
somme <- a + b
moyenne <- somme / 2
```

* Les langages typés nécessitent parfois des conversions explicites (`(REEL)somme`)
* Les langages non typés effectuent des conversions automatiques

---

### **7. Portée et fonctions**

```pseudocode
DEBUT
Fonction calculerDistance(pointA, pointB):
   dx <- pointA.x - pointB.x
   dy <- pointA.y - pointB.y
   distance <- racine_carrée(dx*dx + dy*dy)
   Retourner distance
FIN
```

* `dx`, `dy` et `distance` : **portée locale**, accessibles uniquement dans la fonction
* Limite les interférences et sécurise les données

---

### **8. Essentiel**

* Les variables peuvent être **typées ou non typées**.
* Le **typage** affecte : sécurité, performance, lisibilité et maintenance.
* La **portée** détermine où une variable peut être utilisée : locale (préférée) ou globale.
* Bien gérer types et portées permet de **réduire les erreurs, améliorer la sécurité et la maintenance du code**.
