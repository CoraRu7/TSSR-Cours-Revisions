### **Titre : Tester des systèmes de protection d’accès à Internet**

**Objectif :**
Comprendre et appliquer la méthodologie de test des systèmes de protection d’accès à Internet afin d’identifier les failles, mesurer la performance et renforcer la sécurité globale des infrastructures.

---

### **Concepts clés :**

1. **Systèmes de protection d’accès à Internet :**

   * Ensemble de dispositifs, logiciels et stratégies pour sécuriser les connexions réseau.
   * Objectifs : confidentialité, intégrité, disponibilité des données.
   * Types principaux :

     * **Pare-feux (firewall)** : filtrage du trafic.
     * **IDS/IPS** : détection et prévention d’intrusions.
     * **Proxy** : intermédiaire filtrant et anonymisant les requêtes.
     * **Filtrage de contenu** : blocage de sites ou données non autorisés.
     * **VPN** : connexions sécurisées pour utilisateurs distants.
     * **DLP** : prévention des fuites de données.
     * **Filtrage mail** : anti-spam et sécurité des pièces jointes.
     * **2FA** : authentification à deux facteurs.
     * **Anti-DDoS** : protection contre attaques par déni de service.
     * **Filtrage DNS** : contrôle des résolutions DNS via listes rouges/vertes.

2. **Équilibre sécurité/connectivité :**

   * Sécuriser sans nuire à la productivité.
   * Les professionnels doivent gérer compromis entre protection et performance.

3. **Méthodes de test :**

   * **Pentest (test de pénétration)** : simule une attaque pour exploiter les vulnérabilités.
   * **Tests d’intrusion** : évaluation des IDS/IPS et comportement humain.
   * **Tests de vulnérabilité** : identification des CVE via scanners (OpenVAS, Nessus).
   * **Tests de performance** : évaluer la résilience et disponibilité sous charge.

4. **Outils principaux :**

   * OpenVAS (GVM), Nessus, Wireshark, Nmap, Metasploit, BurpSuite, Snort.

5. **Scénarios et techniques de test :**

   * **Scénarios :** simulations réalistes (ex : tentative d’accès externe à des données sensibles).
   * **Techniques :**

     * Ingénierie sociale, tests d’injection, analyse de vulnérabilités automatisée, pentests.
   * **Mesures de performance :** latence, bande passante, throughput, disponibilité, scalabilité, tolérance aux pannes, tests de charge.

6. **Méthodologie de pentest :**

   * **Planification & préparation** : objectifs, scope, outils, équipe.
   * **Reconnaissance** : collecte d’informations sur la cible.
   * **Identification des vulnérabilités** : analyse approfondie.
   * **Exploitation** : tentative de pénétration.
   * **Analyse des résultats** : impacts et recommandations.
   * **Rapport & présentation** : documentation et préconisations.

7. **Pratique avec OpenVAS (GVM) sous Kali Linux :**

   * Installer : `apt-get update && apt-get upgrade` → `apt-get install gvm` → `gvm-setup` → `gvm-check-setup` → `greenbone-feed-sync` → `gvm-start`.
   * Connexion : `admin` + mot de passe récupéré.
   * Maintenir la base de vulnérabilités à jour.
   * Maîtriser les fonctionnalités avant d’explorer d’autres outils.

### **Commandes/Notes importantes :**

```bash
# Mise à jour système
apt-get update && apt-get upgrade

# Créer dossier pour GVM
mkdir /home/gvm && cd /home/gvm
chmod og+rX /home/ /home/gvm

# Installer GVM (OpenVAS)
apt-get install gvm
gvm-setup
gvm-check-setup
greenbone-feed-sync

# Lancer l’interface web
gvm-start
```

* Login : `admin` / mot de passe récupéré.
* Lire la documentation (RTFM) avant toute utilisation avancée.
* Approfondir un outil avant de passer au suivant.
