# **Le rôle des VLANs**

## **I) Introduction**

### **Contexte**

* Avant les VLANs : un seul **domaine de broadcast** sur le LAN → tous les équipements étaient dans le même groupe.
* Problème : séparation physique nécessaire pour isoler les services (ex : comptabilité, RH).
* Avec les VLANs : **séparation logique** sur le même switch → trafic isolé sans matériel supplémentaire.

### **Fonctionnement d’un LAN**

* Chaque interface hôte : **adresse MAC unique**.
* Switch : table de commutation (MAC Address Table) → associe ports et adresses MAC.
* Types de trafic :

  * **Unicast** : vers un seul hôte
  * **Broadcast** : vers tous les hôtes
  * **Multicast** : vers certains hôtes
* Switch : transfert unicast via le bon port, diffuse broadcast/multicast sur tous les ports sauf origine.
* VLAN = **LAN logique** sur une infrastructure physique commutée.

### **SPB (Shortest Path Bridging)**

* Permet multi-chemins Layer 2.
* Virtualise la connectivité et optimise la restauration après panne.
* Gestion unicast et multicast intégrée.

## **II) Rôle des VLANs et norme IEEE 802.1Q**

### **Définition**

* VLAN = **Virtual LAN**, segmentation logique du réseau.
* **IEEE 802.1Q** : ajout d’une **étiquette VLAN** dans la trame Ethernet (4 octets, 12 bits pour ID VLAN).
* Permet de :

  * **Isoler le trafic** entre différents groupes (ex : utilisateurs, imprimantes).
  * **Réduire le domaine de broadcast**.
  * Appliquer des **ACL** pour contrôler la communication inter-VLAN.

### **Matériel**

* Switch professionnel nécessaire pour configurer plusieurs VLANs.
* VLAN → chaque VLAN = **sous-réseau séparé** (ex : 192.168.10.0/24, 192.168.20.0/24).

---

## **III) Modes de ports sur Cisco**

| **Mode** | **Fonction**              | **Remarques**                                                          |
| -------- | ------------------------- | ---------------------------------------------------------------------- |
| Access   | Trafic d’un seul VLAN     | Connecte hôte ou serveur, pas d’étiquette VLAN ajoutée                 |
| Trunk    | Trafic de plusieurs VLANs | Connecte switch, routeur ou serveur 802.1Q, ajoute des étiquettes VLAN |

* **VLAN natif** : VLAN pour lequel aucune étiquette n’est ajoutée sur un port trunk.

---

## **IV) Types de VLANs**

| **Type**            | **Description**                                                                                  |
| ------------------- | ------------------------------------------------------------------------------------------------ |
| VLAN 1 (par défaut) | VLAN présent sur tous les ports, protocole couche 2 (CDP, VTP, PAgP, DTP) obligatoire            |
| VLAN réservé        | VLANs non exploitables (1002–1005 pour FDDI/token ring)                                          |
| VLAN utilisateur    | VLAN classique pour la segmentation logique des utilisateurs                                     |
| VLAN de gestion     | VLAN pour accéder à la gestion du switch (avec IP/masque)                                        |
| VLAN natif          | Port trunk 802.1Q : trafic non balisé placé ici                                                  |
| VLAN vocal (voice)  | Trafic téléphonique IP, prioritaire, configuré sur port access avec mini-trunk vers téléphone IP |

**Conseils :**

* Modifier VLAN natif = appliquer sur tous les trunks du réseau.
* Priorité au VLAN vocal pour garantir qualité d’appel.

---

## **V) Points clés**

* **VLANs = LAN logique** → séparation du trafic sur le même switch.
* **Segmentation** : sécurité, isolation des groupes et réduction du broadcast.
* **Port access** = 1 VLAN / hôte ; **Port trunk** = plusieurs VLANs / switch.
* **Types VLAN** : par défaut, utilisateur, gestion, natif, vocal.
* **Norme IEEE 802.1Q** : balisage VLAN dans la trame Ethernet.
* VLAN ↔ sous-réseau IP pour la communication entre équipements.
