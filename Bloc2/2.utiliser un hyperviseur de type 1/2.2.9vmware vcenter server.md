
# Fiche de révision – VMware vCenter Server

## Objectif

Comprendre le rôle et l’intérêt de **vCenter Server** dans un environnement VMware et savoir l’installer à partir d’un **fichier ISO** ou **OVA**.

---

## Concepts clés

### Rappels

* **Fichier OVA** : archive contenant tous les fichiers nécessaires pour déployer une VM préconfigurée.
* **Hyperviseur** :

  * **Type 1** : s’exécute directement sur le matériel (ESXi, Hyper-V).
  * **Type 2** : s’exécute sur un OS hôte (Workstation, VirtualBox).
* **Appliance** : VM préconfigurée pour une tâche spécifique (ex. vCenter Server Appliance).

### vCenter Server

* **Définition** : outil centralisé pour gérer plusieurs hyperviseurs, machines virtuelles et objets d’inventaire.
* **Dépendance** : nécessite une base de données (PostgreSQL dans VCSA).
* **Composants principaux** : Photon OS, vSphere Authentication, Lifecycle Manager.

---

## Avantages / Inconvénients

### Avantages

* Centralisation de la gestion.
* Automatisation des tâches répétitives.
* Supervision avancée en temps réel.
* Évolutif, adapté à tous types d’infrastructures.

### Inconvénients

* Coût élevé des licences.
* Complexité d’installation et de configuration.
* Exigences matérielles importantes.
* Dépendance à VMware.
* Courbe d’apprentissage pour les administrateurs.

---

## Architecture et composants

| Composant              | Fonction                                                   |
| ---------------------- | ---------------------------------------------------------- |
| vMotion                | Migration à chaud des VM entre hôtes.                      |
| Storage vMotion        | Migration des fichiers VM entre datastores.                |
| HA (High Availability) | Redémarrage automatique des VM en cas de panne d’un hôte.  |
| DRS                    | Équilibrage automatique de charge entre hôtes.             |
| SDRS                   | Répartition automatique du stockage.                       |
| SIOC / NIOC            | Surveillance et contrôle des performances stockage/réseau. |
| vSwitch distribué      | Commutateur virtuel avancé multi-hôtes.                    |
| SSO / PSC              | Authentification centralisée multi-domaines.               |
| Clone / Backup / FT    | Sauvegarde, clonage et tolérance aux pannes.               |
| DPM                    | Économie d’énergie via arrêt/redémarrage des hôtes.        |
| VUM                    | Gestion des mises à jour ESXi et VM.                       |

## Installation et configuration

### Prérequis

* Respecter **config matérielle minimale** selon taille d’infra et version (CPU, RAM, stockage).
* Vérifier **compatibilité hardware** avec VMware Compatibility Guide.

### Méthodes d’installation

1. **Fichier OVA** (VM préconfigurée)

   * Importer dans VMware Workstation ou ESXi.
   * Configurer réseau (IPv4 statique recommandé).
   * Définir mot de passe root (F2 sur VM) et SSO (administrator@domain).
   * Accéder à la gestion via `https://IP:5480` pour finaliser l’installation.

2. **Fichier ISO**

   * Monter l’ISO sur l’hyperviseur ESXi.
   * Lancer `installer.exe` (Windows) pour déployer VCSA.
   * Définir nom VM, mot de passe root, taille déploiement (tiny/small/etc.).
   * Configurer réseau, IP, gateway, DNS.
   * Compléter configuration SSO et domaine.
   * Accéder à vCenter via navigateur (`https://IP`) et installer les mises à jour.

### Gestion des utilisateurs et privilèges

* **Utilisateurs par défaut** : root (système), administrator@domain (interface web).
* **Méthodologie** :

  1. Créer utilisateurs.
  2. Créer groupes et affecter utilisateurs.
  3. Créer rôles et définir privilèges.
  4. Associer groupes/rôles aux objets d’inventaire.
* **Bonnes pratiques** :

  * Utiliser groupes plutôt qu’utilisateurs.
  * Limiter les privilèges au strict nécessaire.
  * Définir rôles sur objets de haut niveau pour propager.

### Politique de mot de passe

* 8 à 20 caractères, majuscule, minuscule, chiffre et caractère spécial (& # %).

## Cas d’usage

* Entreprises (gestion centralisée de plusieurs hôtes).
* Datacenters virtuels.
* Cloud privé.
* Environnements de développement/test.

---

## L’essentiel

* **vCenter Server** simplifie et centralise la gestion VMware.
* Offre **haute disponibilité, sécurité, performance et automatisation**.
* Installation correcte et configuration initiale sont critiques pour la sécurité et la performance.
