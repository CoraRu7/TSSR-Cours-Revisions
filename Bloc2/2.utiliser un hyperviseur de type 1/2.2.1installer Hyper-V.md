# Fiche – Installer Hyper-V

## 1. Contexte et définition

* **Hyper-V** : hyperviseur de type 1 développé par Microsoft.
* Permet de créer et gérer des **machines virtuelles (VM)** sur un serveur physique.
* Hyperviseur = logiciel qui exécute des VM et gère les systèmes d’exploitation invités.
* Hypercall : interface entre VM et hyperviseur pour opérations privilégiées.
* Hyper-V s’intègre à **Windows Server** ou existe en version autonome **Hyper-V Server**.

### Types de virtualisation

1. **Virtualisation de serveurs** : plusieurs OS sur un même serveur physique.
2. **Virtualisation réseau** : exécution d’un programme sur un réseau virtuel simulé.
3. **Virtualisation de postes de travail** : postes fournis en tant que service.

> Hyperviseur de type 1 : se connecte directement au matériel pour créer des environnements sécurisés.

---

## 2. Avantages d’Hyper-V

* Déploiement rapide de nouvelles VM.
* Maintenance sans temps d’arrêt.
* Migrations en direct simplifiées.
* Sauvegardes facilitées.
* Sécurité intégrée via **Active Directory**.
* Prix attractif (version gratuite disponible).
* Supporte Windows et plusieurs distributions Linux : CentOS, Red Hat, Debian, Ubuntu, SUSE, Oracle Linux, FreeBSD.

### Inconvénients

* La **couche de contrôle** nécessite un OS installé.
* Si la couche de contrôle tombe en panne → toutes les VM tombent.
* Dépendance aux mises à jour de sécurité de Microsoft.
* Plus de frais généraux pour la couche de contrôle.

---

## 3. Coûts

* Dépend du nombre de processeurs et des OS gérés.
* Licences : gratuites jusqu’à 3 607 $ selon versions et besoins.

---

## 4. Performances

* Comparé à VMware, Hyper-V présente une **bonne performance** et un **temps d’initialisation réduit**.
* Bien adapté aux environnements hybrides (cloud privé + cloud public).

---

## 5. Préparation à l’installation

* Vérifier la **configuration matérielle** : support de la virtualisation activé dans le BIOS.
* Vérifier que **VirtualBox ou un autre hyperviseur n’occupe pas la virtualisation**.
* Mettre à jour Windows Server avant l’installation.
* Versions d’évaluation Windows Server : 180 jours gratuites.

---

## 6. Installation d’Hyper-V (GUI sur Windows Server)

1. Ouvrir **Gestionnaire de serveur** → « Ajouter des rôles et fonctionnalités ».
2. Page **Avant de commencer** → « Suivant ».
3. **Type d’installation** : « Basé sur un rôle ou une fonctionnalité » → « Suivant ».
4. Sélectionner le serveur → « Suivant ».
5. Sélectionner **Hyper-V** → « Suivant ».
6. Ajouter les fonctionnalités recommandées → « Suivant ».
7. Confirmer les options par défaut → plusieurs « Suivant ».
8. Confirmer l’installation → « Installer ».
9. Terminer → « Fermer ».
10. Vérifier l’installation : ouvrir **Gestionnaire Hyper-V**.

---

## 7. Points complémentaires

* **Hyper-V Server** : version gratuite, en ligne de commande, adaptée aux expérimentés.
* Peut être utilisé pour **virtualisation, cloud hybride et environnements multi-OS**.
* Conçu pour **sécurité et évolutivité** avec un minimum de frais de maintenance.
