# Fiche de révision – IDS/IPS

**Objectif :** Comprendre l’architecture, les composants, les modes de déploiement et l’intégration des systèmes IDS/IPS pour la sécurité réseau.

---

## Concepts clés

### 1. Rôle d’un IDS/IPS

* **IDS (Intrusion Detection System)** : détecte les intrusions et génère des alertes sans bloquer le trafic.
* **IPS (Intrusion Prevention System)** : détecte et bloque les intrusions en temps réel, action proactive.
* **But** : protéger les réseaux et systèmes informatiques contre les menaces.

### 2. Composants principaux

* **Capteurs** : surveillent le trafic réseau et collectent des données.
* **Analyseurs de données / moteur de détection** : comparent les données avec des signatures et modèles comportementaux pour identifier les anomalies.
* **Base de signatures** : contient les schémas connus d’attaques et de comportements malveillants.
* **Gestionnaire d’alertes** : priorise et notifie les incidents.
* **Système de réponse automatisée (IPS)** : bloque ou limite les intrusions détectées.
* **Tableau de bord et interface de gestion** : surveille, configure et consulte les alertes.

### 3. Modes de déploiement

* **Inline** : IDS/IPS dans le chemin du trafic → détection et blocage en temps réel.

  * ✅ Blocage actif, réponse rapide, protection robuste
  * ❌ Faux positifs possibles, impact performance
* **Monitor** : IDS/IPS en écoute passive → uniquement alertes

  * ✅ Pas de perturbation, moins de faux positifs
  * ❌ Intervention humaine nécessaire, moins proactif
* **Promiscuous** : IDS/IPS observe le trafic sans être sur le chemin

  * ✅ Surveillance passive
  * ❌ Intervention humaine nécessaire, pas de blocage actif

### 4. Actions préventives et curatives

* **Préventives** :

  * Mise à jour régulière des signatures
  * Segmentation du réseau
  * Surveillance continue du trafic et des journaux
  * Formation des utilisateurs
* **Curatives** :

  * Alertes en temps réel et réaction rapide
  * Analyse post-incident et révision des politiques de sécurité
  * Mise en quarantaine des systèmes compromis

### 5. Intégration avec d’autres systèmes

* **Pare-feu** : coordination pour bloquer ou rediriger le trafic suspect.
* **SIEM (Security Information and Event Management)** : collecte et corrèle les événements pour analyser et répondre aux incidents.
* **Antispam** : complémentaire, IDS/IPS peut détecter phishing ou malware via e-mails.

### 6. Solutions populaires

* **Open Source** : Snort, Suricata
* **Propriétaire / matériel dédié** : Cisco Firepower
* **Modes de fonctionnement** : logiciel sur serveur ou VM, ou solution matérielle.

## Commandes / Notes pratiques (exemple Snort sur Linux)

* Mise à jour système :

  ```bash
  sudo apt-get update
  sudo apt-get upgrade
  ```
* Installation Snort :

  ```bash
  sudo apt-get install snort
  ```
* Configuration réseau : identifier interface active

  ```bash
  ifconfig
  ```
* Configuration règles personnalisées :

  ```bash
  sudo nano /etc/snort/rules/local.rules
  alert icmp any any -> $HOME_NET any (msg:'tentative connection icmp'; sid:00001; rev:1;)
  ```
* Lancer Snort :

  ```bash
  sudo snort -A console -i enp0s3 -U snort -c /etc/snort/snort.conf
  ```
* Lancer en tâche de fond : créer un service systemd pour Snort

  ```bash
  sudo nano /lib/systemd/system/snort.service
  sudo systemctl status snort
  ```
